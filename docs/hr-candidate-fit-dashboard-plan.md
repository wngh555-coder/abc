# 적임자 선발 대시보드 기획서 (Streamlit)  
**용도**: 유학·주재원·해외 파견·글로벌 프로젝트 등 **선발 분야(트랙)별**로 HR이 후보를 비교·검토할 때 참고하는 내부용 데이터 화면

## 1. 문서 목적

- 회사 **HR / 인재개발 / 글로벌모빌리티** 담당자가 동일 기준으로 후보를 훑고, **분야별 요건과의 부합도**를 한 화면에서 정리할 수 있는 Streamlit 대시보드의 범위·데이터·화면·운영 원칙을 정의한다.
- 본 문서는 **의사결정 보조(참고)** 도구이며, **최종 선발의 법적·정책적 책임**은 회사 규정과 담당 조직에 있음을 전제로 한다.

## 2. 제품 한 줄 정의

> 선발 분야(유학 파견, 주재원, 단기 프로젝트 등)를 고르면 **요건·후보 프로필·(선택) 점수화 결과**가 필터·표·차트로 정리되고, HR이 **동일 기준으로 후보를 비교**할 수 있는 Streamlit 기반 내부 대시보드.

## 3. 목표 및 비목표

| 구분 | 내용 |
|------|------|
| **목표** | 분야별 필수/우대 요건 가시화, 후보 리스트 탐색·정렬·비교, 근거 필드(어학·기간·직무·가족 동반 등) 빠른 확인 |
| **비목표(초기)** | 외부 공개 채용 사이트, 법적 효력 있는 “자동 합격/불합격” 단정, 개인정보 무단 수집·제3자 제공 |

## 4. 사용자·시나리오

- **HR 담당**: 이번 분기 주재원 후보 풀을 “언어·가족·건강 적합(자가신고)” 기준으로 1차 스크리닝 후 면담 대상만 추린다.
- **인재개발**: 유학·연수 트랙별 정원 대비 지원 가능 인원을 파악하고, 부서장에게 후보군을 공유한다(내부 정책에 따른 비식별/최소 공개).
- **글로벌모빌리티**: 국가·체류 기간·동반 가족 조건이 다른 포지션별로 **적합 후보 후보군**을 나란히 본다.

## 5. 선발 분야(트랙) 정의

### 5.1 트랙 예시 (회사 정책에 맞게 확장)

| 트랙 ID | 예시 명칭 | 특징적 검토 축 (예시) |
|---------|-----------|------------------------|
| `study_abroad` | 유학·연수 파견 | 학위/과정 기간, 복귀 의무, 언어, 학비·지원 한도 |
| `expat` | 장기 주재원 | 파견 기간, 가족 동반, 현지 급여·복지, 언어, 비자 적격 |
| `short_term` | 단기 프로젝트 | 출장 빈도, 체류 일수, 업무 연속성 |
| `global_rotation` | 글로벌 로테이션 | 순환 주기, 이전 해외 경험, 직무 매칭 |

- 각 트랙은 **요건 프로파일**(필수/우대/제외)을 데이터로 가진다.

### 5.2 요건 차원(평가 축) 예시

- **언어**: 목표 언어·레벨(예: 영어 OPIc/TOEIC 등 **회사 표준 스키마**)
- **직무**: 직무군·필수 경력 연수·핵심 스킬 태그
- **가족·동반**: 동반 가능 여부, 자녀 학교 이슈(자가신고/내부 기록)
- **건강·적응**: 출장 적합(자가신고), 면담/검진 결과는 **별도 시스템 연동 또는 수기 입력** (민감정보 취급 주의)
- **규정**: 복귀 의무, 재파견 제한, 징계 이력 유무(접근 권한 분리)

※ 실제 구현 시 **개인정보·고유식별·건강정보**는 법적 근거·목적 제한·보관 기간을 반드시 설계한다.

## 6. 데이터 모델(논리 스키마)

| 엔티티 | 필드 예시 | 설명 |
|--------|-----------|------|
| **Employee** | `employee_id`, `name`(또는 비식별 ID), `dept`, `job_family`, `tenure_months` | 후보 마스터 |
| **LanguageSkill** | `employee_id`, `lang`, `test_type`, `score`, `valid_until` | 어학 |
| **AssignmentHistory** | `employee_id`, `country`, `start`, `end`, `role` | 해외·파견 이력 |
| **TrackRequirement** | `track_id`, `dimension`, `rule_type`(필수/우대/제외), `threshold`, `weight`(선택) | 트랙별 요건 |
| **FitScore** *(선택)* | `employee_id`, `track_id`, `score`, `breakdown_json`, `computed_at` | 사전 계산 또는 온디맨드 |
| **AuditLog** *(권장)* | `viewer_id`, `action`, `timestamp`, `filters` | 열람·다운로드 기록 |

- **원천 데이터**는 CSV/DB/API 중 회사 환경에 맞게 선택; MVP는 **익명화된 CSV**로도 검증 가능.

## 7. 정보 구조(대시보드 IA)

```
헤더: 솔루션명, 데이터 기준일, “참고용·최종 판단은 내부 규정” 배너
사이드바
  ├── 선발 트랙 선택 (유학 / 주재원 / 단기 / …)
  ├── 부서·직무군·경력 구간 필터
  ├── 언어 조건 (언어 + 최소 점수/등급)
  ├── 해외 경험 유무·기간
  ├── (선택) 가족 동반 필요 여부
  └── 요건 엄격도: “필수만 충족” / “우대 반영”
메인
  ├── KPI: 트랙별 후보 수 / 필수 충족 인원 / 평균(중앙) 어학 등
  ├── 탭1: 후보 목록 (정렬·검색·컬럼 선택) + 행 클릭 시 상세(expand)
  ├── 탭2: 비교 뷰 — 2~5명 선택 시 레이더/평행좌표/바 차트
  ├── 탭3: 요건 충족 요약 — 차원별 충족률·미달 사유(필드 단위)
  └── (선택) 지도/국가별 분포 — 파견 가능 국가와 매칭 시
푸터/Expander: 데이터 정의, 갱신 주기, 개인정보·차별금지·이의제기 절차 안내
```

## 8. 기능 요구사항

### 8.1 필수(MVP)

- 트랙 선택 시 **해당 트랙 요건**이 화면에 노출(필수/우대 구분).
- 후보 **테이블**: 필터·정렬·텍스트 검색.
- **필수 요건 미충족** 후보는 시각적 구분(행 스타일 또는 플래그 컬럼).
- 데이터 **기준일·출처** 표시.
- Streamlit `st.session_state`로 비교 선택(체크박스) 유지.

### 8.2 권장

- **적합도 점수**(가중치는 HR이 조정 가능한 설정 파일 또는 사이드바 슬라이더) — **설명 가능성**(breakdown) 병기.
- **CSV보내기**: 권한 있을 때만, 비식별 옵션.
- **감사 로그** 또는 최소한 “다운로드 시각·필터” 로컬 기록.

### 8.3 후순위

- LDAP/HRIS API 연동, 승인 워크플로, 면담 노트 연계.
- 다국어 UI, 역할 기반 접근(RBAC).

## 9. 시각화·지표 정의

| 위젯 | 데이터 | 비고 |
|------|--------|------|
| 트랙 요건 패널 | `TrackRequirement` | 필수는 빨강/우대는 파랑 등 색 구분 |
| 후보 표 | `Employee` + 조인 | 민감 컬럼은 role에 따라 숨김 |
| 레이더/평행좌표 | 정규화된 차원 점수 | “설명 가능한 AI”가 아니라 **규칙 기반 점수** 권장 |
| 충족률 바 | 차원별 % | 집단 수준 요약 |

## 10. 준법·윤리·품질 원칙

- **차별 금지**: 성별·연령·출신 등 **법으로 보호되는 속성**을 점수에 직접 반영하지 않는다(목적 외 이용 금지).
- **투명성**: 후보에게 **어떤 항목이 반영되는지** 내부 가이드와 일치.
- **정확성**: 자가신고 vs 검증 데이터 구분 표시.
- **최소 수집**: 대시보드에 필요한 필드만 로드.
- **문구**: 화면 상단에 **“자동 선발 아님 / 참고용”** 고정 노출.

## 11. 기술 스택(프로젝트 정합)

- **UI**: Streamlit  
- **데이터**: pandas, (선택) SQLAlchemy  
- **차트**: Plotly  
- **설정**: 트랙·요건 YAML/JSON, `.streamlit/secrets.toml`로 연결 정보  
- **실행**: `streamlit run hr_candidate_dashboard.py` (구현 시 파일명 조정)

## 12. 구현 단계

1. **MVP**: 샘플 CSV + 단일 트랙(주재원) + 필터 + 표 + 필수조건 플래그.  
2. **v1.1**: 복수 트랙, 비교 탭, 규칙 기반 점수+breakdown, CSV export.  
3. **v1.2**: 권한·감사·API 연동.
