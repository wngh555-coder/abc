# 선택 기반 인생 시뮬레이터 · 데이터 대시보드 기획서 (Streamlit)

## 1. 문서 목적

- **선택(분기)에 따라 능력치·자원·관계가 변하는 인생 시뮬레이션 게임**을 Streamlit으로 구현할 때의 범위, 게임 설계, 데이터 구조, 대시보드 IA, 구현 단계를 정의한다.
- 플레이 본체(스토리·선택)와 **지표·타임라인·분기 결과를 한눈에 보는 데이터 대시보드**를 동일 앱 안에서 연결하는 것을 목표로 한다.

## 2. 제품 한 줄 정의

> 플레이어가 연도(또는 턴)마다 제한된 선택지를 고르면 **건강·재력·관계·커리어** 등 지표가 변하고, 그 변화가 **차트·타임라인·이벤트 로그**로 시각화되는 Streamlit 기반 텍스트·선택형 라이프 시뮬레이터.

## 3. 목표 및 비목표

| 구분 | 내용 |
|------|------|
| **목표** | 명확한 선택 UI, 상태(스탯)의 원인·결과 추적, 턴별 스냅샷으로 **인생 곡선** 시각화, 저장/불러오기 |
| **비목표(초기)** | 3D·애니메이션, 음성·이미지 대량 에셋, 멀티플레이·서버 동기화, 외부 LLM 실시간 내러티브(후순위로 옵션화 가능) |

## 4. 사용자·시나리오

- **캐주얼 플레이어**: 짧은 시나리오(10~30턴)로 끝까지 한 번 밀고, 마지막에 “내 인생 그래프”를 본다.
- **실험형 플레이어**: 같은 분기에서 다른 선택만 바꿔 **통계 차이**를 비교한다(세션 A/B 또는 리플레이).
- **교육·워크숍**: “선택이 장기 지표에 미치는 영향”을 토의 자료로 대시보드를 띄운다.

## 5. 게임 루프(핵심 설계)

### 5.1 턴 구조

1. **상태 표시**: 현재 연도(또는 나이), 핵심 스탯 요약.
2. **상황 텍스트**: 짧은 에피소드 설명(1~3문단).
3. **선택지**: 2~4개 버튼(또는 라디오). 각 선택은 `choice_id`로 식별.
4. **결과 적용**: 선택에 매핑된 `effects`를 스탯에 가산(클램프·임계값 규칙 적용).
5. **기록**: 타임라인에 `{turn, choice_id, narrative_snippet, stats_snapshot}` 추가.
6. **종료 조건**: 최대 턴 도달, 특정 스탯 임계(사망/은퇴/목표 달성) 등.

### 5.2 스탯(예시 · 기획에서 확정)

| 스탯 | 의미 | 범위 예시 |
|------|------|-----------|
| `health` | 건강 | 0~100 |
| `wealth` | 재력/자산 지수 | 0~100 또는 실수 누적 |
| `happiness` | 주관적 만족 | 0~100 |
| `career` | 직업·성취 | 0~100 |
| `relationship` | 관계(가족·친구) | 0~100 |

- 초기값은 시나리오별로 정의(예: 전원 50).
- **클램프**: 모든 스탯을 `[min, max]`로 제한해 대시보드 스케일을 고정한다.

### 5.3 콘텐츠 모델(데이터 주도)

- **시나리오**: JSON/YAML 또는 Python dict로 정의 — `nodes`(턴/상황)와 `choices`(다음 노드 또는 효과).
- **노드 필드 예**: `id`, `title`, `body`, `choices[]`, `is_terminal`.
- **선택 필드 예**: `id`, `label`, `effects` (스탯 델타), `next_node_id`, `unlock_flags`(후순위).

이렇게 하면 코드 수정 없이 **콘텐츠만 추가**해 시나리오를 늘릴 수 있다.

## 6. 데이터 모델(논리 스키마)

| 엔티티 | 필드 예시 | 설명 |
|--------|-----------|------|
| **GameState** | `turn`, `age`, `node_id`, `stats` | 현재 진행 |
| **Stats** | 위 스탯 키들 | 턴마다 스냅샷 |
| **TimelineEntry** | `turn`, `node_id`, `choice_id`, `stats_after`, `ts` | 대시보드 시계열 소스 |
| **RunMeta** | `scenario_id`, `started_at`, `ended`, `ending_id` | 한 판 메타 |

**집계·파생**: 턴별 스냅샷으로 라인 차트; 마지막 턴 기준 레이더(스탯 균형); 선택 빈도(동일 세션 내).

## 7. 정보 구조(대시보드 IA)

```
헤더: 시나리오명, 현재 턴/나이, 진행률(턴/최대턴)
┌─ 메인 좌측(또는 상단): 게임 패널 ─────────────────┐
│  상황 텍스트                                      │
│  [선택 A] [선택 B] [선택 C]                       │
│  (종료 시) 엔딩 요약 + 다시하기                    │
└──────────────────────────────────────────────────┘
┌─ 메인 우측(또는 하단): 데이터 대시보드 ───────────┐
│  KPI: 주요 스탯 카드 또는 게이지                  │
│  탭1: 스탯 추이 (멀티 라인 / 면적)                │
│  탭2: 타임라인 테이블 (턴·선택·스탯)              │
│  탭3: 최종/현재 스탯 레이더 또는 바 비교          │
│  [선택] 분기별 평균·카운트(고급)                  │
└──────────────────────────────────────────────────┘
사이드바
  ├── 시나리오 선택(프리셋)
  ├── 새 게임 / 이어하기(세션)
  ├── JSON보내기·가져오기
  └── 난이도·최대 턴(옵션)
푸터/Expander: 실험용 시뮬레이터, 현실 인생과 무관함 등 안내
```

Streamlit 특성상 **한 페이지에서 게임 + 차트**를 세로 스크롤 또는 `st.columns`로 배치하면 구현이 단순하다.

## 8. 기능 요구사항

### 8.1 필수(MVP)

- 최소 **1개 시나리오**, **10턴 전후** 분기 구조.
- 선택 후 스탯 갱신·타임라인 append·다음 노드 이동.
- **스탯 추이 라인 차트**(Plotly) + **타임라인 `st.dataframe`**.
- `st.session_state`에 `GameState` 보관, **초기화(새 게임)** 버튼.
- 불가능한 선택(정의상 없음) 방지 — UI는 현재 노드의 선택지만 노출.

### 8.2 권장

- 진행 상황 **JSON 저장/불러오기**(가상 투자 앱과 동일 패턴).
- 종료 시 **엔딩 카드**(텍스트 + 최종 스탯).
- 스탯 **클램프·경고**(0 이하 시 “건강 위기” 등 이벤트 노드로 강제 분기).

### 8.3 후순위

- 복수 시나리오·업적·수집 요소.
- 간단한 **랜덤 이벤트**(확률 테이블).
- LLM으로 상황 문장 생성(옵션, 비용·지연 고려).

## 9. 시각화·지표 정의

| 위젯 | 데이터 | 비고 |
|------|--------|------|
| 스탯 추이 | `TimelineEntry.stats_after` 시계열 | x=턴 또는 나이 |
| 레이더 | 마지막 스냅샷 5축 | Plotly `go.Scatterpolar` |
| 타임라인 표 | 턴, 선택 라벨, 스탯들 | 정렬·다운로드 CSV |
| 진행률 | `turn / max_turn` | `st.progress` |

## 10. 기술 스택(프로젝트 정합)

- **UI**: Streamlit (`st.button` / `st.radio` / `st.session_state`)
- **데이터**: pandas(타임라인 DataFrame 변환)
- **차트**: Plotly
- **콘텐츠**: JSON 파일 또는 `scenario_*.py` 상수
- **실행**: `streamlit run life_sim_app.py` (구현 시 파일명은 프로젝트에 맞게)

## 11. 구현 단계

1. **MVP**: 단일 시나리오, 노드 8~12개, 스탯 3~5개, 라인+표, 세션 상태.  
2. **v1.1**: JSON 저장/불러오기, 엔딩 분기 2종 이상, 레이더 차트.  
3. **v1.2**: 시나리오 2편, 랜덤 이벤트, 스탯 임계로 강제 노드.  

## 12. 리스크·대응

| 리스크 | 대응 |
|--------|------|
| 분기 누락·순환 참조 | 시나리오 로드 시 `next_node_id` 검증 스크립트 또는 assert |
| 스트림릿 버튼 중복 실행 | 선택 처리는 한 함수로 묶고, 처리 후 `st.rerun()` 또는 상태 플래그 |
| 텍스트 과다로 스크롤 부담 | `st.expander`로 과거 로그 접기 |
