# 가상 투자 게임 · 데이터 대시보드 기획서 (Streamlit)

## 1. 문서 목적

- **가상 투자(페이퍼 트레이딩)** 를 주제로 한 Streamlit 대시보드의 범위, 게임 규칙, 데이터 구조, 화면 구성, 구현 단계를 정의한다.
- 실제 증권 주문 없이 **포트폴리오·손익·거래 이력**을 시각화하고, 학습·시뮬레이션 목적에 맞게 설계한다.

## 2. 제품 한 줄 정의

> 사용자가 가상 자금으로 종목을 매수·매도하고, **현재 평가액·누적 손익·자산 배분·거래 내역**을 한 화면에서 확인하는 Streamlit 기반 데이터 대시보드.

## 3. 목표 및 비목표

| 구분 | 내용 |
|------|------|
| **목표** | 직관적인 주문 UI, 명확한 KPI, 시간에 따른 자산 곡선, 보유 종목 및 체결 기록 표시 |
| **비목표(초기)** | 실계좌 연동, 실시간 호가·체결, 법적 투자 자문, 멀티유저 서버·DB 인증 |

## 4. 사용자·시나리오

- **입문자**: 미리 정해진 시나리오(예: NVDA, ETF 2~3개)로 매수/매도를 눌러보며 지표 변화를 익힌다.
- **실험자**: 기간·종목을 바꾸며 가상 포트폴리오의 변동성과 수익률을 비교한다.
- **발표/교육**: 대시보드를 띄운 채로 “가상 자금 1,000만 원으로 시작” 같은 데모를 진행한다.

## 5. 게임 규칙(도메인 설계)

### 5.1 초기 조건

- **시작 현금** (예: 10,000,000원 또는 USD 등 기준 통화 1종).
- **거래 단위**: 정수 주 또는 소수 주(정책 선택; MVP는 정수 권장).
- **수수료·슬리피지**(선택): 0%로 시작 후 v1.1에서 옵션화 가능.

### 5.2 주문

- **매수**: 현금 ≥ (주가 × 수량 \[+ 수수료]) 일 때만 체결.
- **매도**: 보유 수량 ≥ 매도 수량일 때만 체결.
- 체결 가격: **당일 종가** 또는 **마지막으로 조회한 시세**로 단순화(MVP). (실시간 시세는 `yfinance` 등 지연 데이터 전제 명시)

### 5.3 시간 모델

- **실시간 모드**: “지금 시세” 기준 평가 (교육용 단순 모드).
- **과거 시뮬레이션 모드**(후순위): 특정 일자로 되돌려 그 시점 종가로만 거래 허용 등.

### 5.4 상태 저장

- **세션**: `st.session_state`에 현금, 보유, 거래 로그 유지 (브라우저 세션 단위).
- **저장/불러오기**(권장): JSON/CSV로보내기·가져오기 또는 로컬 파일 캐시.

## 6. 데이터 모델(논리 스키마)

| 엔티티 | 필드 예시 | 설명 |
|--------|-----------|------|
| **Portfolio** | `cash`, `currency` | 가용 현금 |
| **Position** | `symbol`, `qty`, `avg_cost` | 종목별 보유 수량·평균 단가 |
| **Trade** | `ts`, `symbol`, `side`, `qty`, `price`, `fee` | 체결 이력 (시계열) |
| **Quote** | `symbol`, `date`, `close`, (open/high/low/volume) | 시세 스냅샷 (캐시된 DataFrame) |

집계 지표는 위에서 파생: **총 평가액** = 현금 + Σ(수량 × 현재가), **누적 손익** = 총 평가액 − 초기 자금 − 순수수료 등.

## 7. 정보 구조(대시보드 IA)

```
헤더: 게임명, 기준 통화, 마지막 시세 갱신 시각(또는 데이터 지연 안내)
사이드바
  ├── 종목 선택 / 심볼 입력(제한 목록 권장)
  ├── 기간(시세 차트용)
  ├── 매수·매도 폼 (수량, 버튼)
  ├── [선택] 초기화 / 시드 머니 재설정
  └── [선택] 상태 저장·불러오기
메인
  ├── KPI 행: 현금 | 총 자산 | 누적 손익(금액·%) | 보유 종목 수
  ├── 탭 또는 2열
  │     ├── 자산 곡선 (시간 vs 총 자산 또는 일별 스냅샷)
  │     ├── 시세 차트 (캔들/라인) + 현재가 표시
  │     └── 자산 배분 (파이/바: 현금 vs 종목별 시가총액 비중)
  ├── 보유 종목 표
  └── 거래 내역 표 (시간 역순)
푸터/Expander: 데이터 출처(yfinance 등), 지연·면책 문구
```

## 8. 기능 요구사항

### 8.1 필수(MVP)

- 시세 조회 및 캐싱 (`st.cache_data`, TTL 명시).
- 매수/매도 후 **즉시** KPI·표·(가능하면) 차트 갱신.
- 빈 보유·잔고 부족·심볼 오류 시 `st.warning` / `st.error`.
- **면책 문구**: 교육용 시뮬레이션, 투자 권유 아님.

### 8.2 권장

- 거래 내역·스냅샷 CSV 다운로드.
- 종목 목록 프리셋(예: `NVDA`, `AAPL`, `SPY`)으로 입력 실수 방지.
- `st.session_state` 구조를 함수로 캡슐화해 테스트 용이성 확보.

### 8.3 후순위

- 수수료·세금 단순 모델.
- 여러 포트폴리오 슬롯.
- 간단한 리더보드(로컬 파일 기준).

## 9. 시각화·지표 정의

| 위젯 | 데이터 | 비고 |
|------|--------|------|
| 총 자산 | 현금 + 포지션 평가 | 현재가는 최신 캐시 시세 |
| 누적 수익률 | (총 자산 − 초기 자금) / 초기 자금 | 통화 일치 전제 |
| 자산 곡선 | 거래일마다 또는 액션마다 스냅샷 누적 | MVP는 “액션 후 포인트”만으로도 가능 |
| 배분 차트 | 종목별 평가액 비중 | Plotly pie/bar |
| 시세 차트 | 선택 종목 OHLC 또는 종가 라인 | 기존 `nvidia_dashboard.py` 패턴 재사용 가능 |

## 10. 기술 스택(프로젝트 정합)

- **UI**: Streamlit  
- **데이터**: pandas  
- **시세**: yfinance(지연 데이터) — 레포 내 `nvidia_dashboard.py`와 동일 계열  
- **차트**: Plotly  
- **상태**: `st.session_state` + 선택적 JSON 직렬화  

## 11. 구현 단계

1. **MVP**: 단일 통화, 단일/복수 종목 프리셋, 매수·매도, KPI, 보유·거래 표, 시세 라인 차트.  
2. **v1.1**: 자산 곡선 스냅샷, CSV 저장/불러오기, 수수료 옵션.  
3. **v1.2**: 과거 일자 시뮬레이션, 다중 포트폴리오.
