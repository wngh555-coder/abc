# 사내 직원 인사정보 통계 대시보드 기획서 (Streamlit)

## 1. 문서 목적

- HR·조직 담당자가 **직원 스냅샷**을 바탕으로 인원 규모·구성·입퇴사·근속 등 통계를 Streamlit에서 확인할 수 있는 범위·데이터·화면·운영 원칙을 정의한다.
- 구현 파일: `hr_employee_dashboard_app.py`, `hr_employee_io.py`, `hr_employee_charts.py`, 샘플 데이터 `data/sample_hr_employees.csv`.

## 2. 제품 한 줄 정의

> 사내 직원 인사 스냅샷을 연결하면 **필터 → KPI → 차트 → 상세 표** 흐름으로 통계를 보여 주는 Streamlit 기반 내부 대시보드(참고용).

## 3. 목표 및 비목표

| 구분 | 내용 |
|------|------|
| **목표** | 조직·직군·고용형태·재직 상태별 집계, 입사 추이·근속 분포 시각화, 필터 결과 CSV보내기 |
| **비목표(초기)** | 급여·평가 원문, 외부 공개, 법적 효력 있는 인사 자동 결정 |

## 4. 사용자·시나리오

- **HR**: 월말 헤드카운트, 정규직 비율, 부서별 인원을 확인한다.
- **조직개발**: 본부·지역·직군 필터로 세그먼트를 비교한다.

## 5. 데이터 모델(샘플 스키마)

| 컬럼 | 설명 |
|------|------|
| `employee_id` | 내부 직원 코드(샘플은 가공) |
| `snapshot_date` | 스냅샷 기준일 |
| `division`, `dept`, `team` | 조직 계층 |
| `location` | 근무 지역 |
| `job_family`, `grade` | 직군·직급 |
| `employment_type` | 정규직/계약직/인턴 등 |
| `status` | 재직/휴직/퇴사 |
| `hire_date`, `termination_date` | 입사·퇴사일(퇴사자만) |

`tenure_months`는 `hire_date`와 스냅샷 기준일로 계산한다.

## 6. 정보 구조(IA)

- **헤더**: 제목, 스냅샷 기준일·출처 캡션  
- **사이드바**: 본부·부서·지역·직군·직급·고용형태·재직상태 필터  
- **메인**: KPI → 조직별 인원 / 직군 분포 / 고용형태 / 입사 추이 / 근속 히스토그램 → 상세 표 + CSV 다운로드  
- **Expander**: 지표 정의, 개인정보·참고용 안내  

## 7. 기술 스택·실행

- Streamlit, pandas, Plotly  
- 실행: `streamlit run hr_employee_dashboard_app.py`

## 8. 지표 정의(합의 권장)

- **대상 인원**: 필터 적용 후 행 수  
- **재직자**: `status == 재직`  
- **평균 근속(월)**: 스냅샷일 − `hire_date` 월 단위 평균(휴직·퇴사 포함 여부는 정책에 따름; 본 샘플은 필터 결과 전체에 대해 계산)
