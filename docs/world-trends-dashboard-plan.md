# 세계 트렌드 실시간 탐지기 — Streamlit 대시보드 기획서

## 1. 문서 목적

- **뉴스·소셜 키워드**를 기준으로 핫 토픽을 한 화면에서 탐지·비교할 수 있는 대시보드의 범위, 화면 구조, 데이터·기술 선택을 정의한다.
- 레포 공통 패턴(`st.set_page_config`, 사이드바 필터, `@st.cache_data`, Plotly)과 맞춘다.

## 2. 배경 및 목표

| 구분 | 내용 |
|------|------|
| **문제** | 글로벌 이슈가 국가·시간대별로 다르게 부각되어, 단일 지표만으로는 맥락 파악이 어렵다. |
| **목표** | 키워드 강도(워드클라우드), **지역별 관심도(지도)**, **시간 축 변화(라인)**를 동일 필터로 연동해 “어디서·언제·무엇이 뜨는지”를 빠르게 파악한다. |
| **비목표(초기 데모)** | 로그인, 유료 소셜 API 전면 연동, 초저지연 실시간 스트리밍 |

## 3. 사용자·시나리오

- **미디어/기획**: 주제 프리셋·기간을 바꾸며 핫 키워드와 지역 편향을 본다.
- **운영 데모**: 갱신 버튼·TTL 캐시로 API 부하 없이 “거의 실시간” 체감을 재현한다.

## 4. 정보 구조(IA)

```
헤더: 제목, 데이터 성격(데모/운영), 갱신 시각
사이드바: 주제 프리셋, 분석 기간(시간), 수동 새로고침
메인
├── KPI: 추적 키워드 수, 피크 시각, 상위 국가(요약)
├── 탭 또는 세로 스택
│   ├── 워드클라우드 (키워드 가중치)
│   ├── 국가별 관심도 (Choropleth, ISO-3166 alpha-3)
│   └── 시간별 트렌드 (라인/영역)
└── (선택) 키워드·점수 요약 표
```

## 5. 데이터 전략

| 단계 | 내용 |
|------|------|
| **데모(현재)** | 시드 기반 시뮬레이션: 주제별 키워드 풀에서 가중치·국가·시계열을 생성. 새로고침 시 시드 변경으로 “스냅샷 갱신” 체감. |
| **운영(확장)** | Twitter/X API, News API, RSS 집계 파이프라인 → 동일 스키마(`word_freq`, `country_df`, `timeline_df`)로 적재 후 동일 UI 재사용. |

## 6. 기능 요구사항

### 6.1 필수

- 워드클라우드: 키워드 빈도(가중치) 시각화, 한글 폰트 경로 처리(Windows `Malgun Gothic` 등).
- 지도: Plotly `choropleth`, `locationmode='ISO-3'`.
- 트렌드: 시간축 라인(또는 영역) 차트, 기간은 사이드바와 연동.
- 캐싱: `@st.cache_data(ttl=…)`로 반복 실행 비용 완화.

### 6.2 권장

- 빈 데이터·폰트 실패 시 `st.warning` / 기본 폰트 폴백.
- 캡션에 “데모 데이터” 명시.

## 7. 기술 스택

- **UI**: Streamlit  
- **차트**: Plotly (지도·시계열), WordCloud + PIL (워드클라우드 래스터)  
- **데이터**: pandas, numpy  

## 8. 실행

```bash
streamlit run world_trends_app.py
```

## 9. 향후 로드맵

- RSS/뉴스 소스 어댑터 모듈 분리  
- 국가 정규화(영문 국명 → ISO-3) 유틸  
- 다국어 불용어·토크나이저(한·영 혼합 텍스트)
